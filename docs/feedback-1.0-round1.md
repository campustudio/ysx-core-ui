## 五、登录流程 [待开发]

### 9. 登录交互流程设计
**反馈**：参考潮汐但有自己的创新。

#### 9.1 未登录状态
- 左上角头像区域显示空白头像图标（无图片）
- 点击空头像 → 弹出登录面板（半屏弹出）

#### 9.2 登录面板（参考潮汐图四）
- 顶部：消息图标 + 设置图标 + 关闭按钮
- 显示「登录元思想」+ 空头像
- 下方入口列表：我的元思想、设置等
- 点击登录相关区域 → 跳转到登录页

#### 9.3 登录页（参考潮汐图五）
- 品牌标识 + 欢迎语
- 自然风景背景图
- 登录选项（从上到下）：
  - 微信登录（主推，醒目按钮）
  - 手机号登录
  - Apple 登录
  - 其它登录方式（折叠入口）
- 底部：隐私协议勾选 checkbox

#### 9.4 隐私协议弹框（参考潮汐图六）
- 未勾选协议时点登录 → 弹出确认框
- 标题：「服务条款与隐私协议」
- 内容：提示用户阅读并同意
- 按钮：取消 / 同意（同意按钮用品牌色高亮）
- 点击「服务条款」或「隐私协议」可跳转查看全文

#### 9.5 已登录状态（参考潮汐图七）
- 左上角头像显示用户头像
- 点击头像 → 弹出个人面板
- 显示用户数据统计（使用天数等）
- 入口：我的收藏、最近播放等

**技术备注**：
- 登录功能需要后端支持（Supabase Auth 或自建）
- 微信登录需要小程序 AppID + 服务端对接
- 当前阶段可先做 UI 外壳 + 模拟交互流程
- TARO 转换时微信登录使用 `wx.login()` API
