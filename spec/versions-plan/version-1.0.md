## 三、技术方案

### 3.1 域名与二维码原理

**关键结论**：

- 二维码 = URL 的图形编码
- 只要域名不变，二维码永久有效
- `https://` 和 `http://` 的二维码不同，统一使用 HTTPS

**域名策略**：

```
https://www.metamindpt.com/  ← 二维码编码此地址
```

- 域名每年续费即可保持有效
- 底层服务器/OSS 可随时更换，只需修改 DNS 指向

### 3.2 SSL 证书方案

| 方案         | 费用      | 有效期 | 推荐       |
| ------------ | --------- | ------ | ---------- |
| 免费证书     | 0         | 3个月  | 需频繁续期 |
| 免费升级版   | 68元/年   | 12个月 | ✅ 推荐    |
| 正式 OV 证书 | 几百~几千 | 1-3年  | 非必要     |

**选择**：68元/年升级版，DV 证书，加密功能与付费证书完全相同。

**证书类型说明**：

- **DV（域名验证）**：验证域名所有权，适合个人/小团队
- **OV（组织验证）**：验证企业资质，显示公司名称
- **EV（扩展验证）**：最高级别，适合金融/电商

我们的场景（内部学员访问）用 DV 完全足够。

### 3.3 ICP 备案

网站底部需展示备案信息（政策要求）：

```
© 2025 元思想（北京）文化传播有限公司
川公网安备 51015602001384号
京ICP备 2025142983号-1
```

备案与证书类型无关，DV 证书可正常展示备案号。

### 3.4 PWA 技术要点

| 要求           | 说明                                  |
| -------------- | ------------------------------------- |
| HTTPS          | PWA 必须使用 HTTPS                    |
| manifest.json  | 定义应用名称、图标、主题色            |
| Service Worker | 实现离线缓存、后台同步                |
| 图标资源       | 需要多尺寸图标（192x192、512x512 等） |

### 3.5 小程序技术要点

| 项目     | 说明                               |
| -------- | ---------------------------------- |
| 账号     | 使用已有企业账号                   |
| 框架     | **原生微信小程序**（不使用 Taro）  |
| 备案     | 需要单独的小程序备案（同事跟进中） |
| 审核     | 需通过微信审核才能发布             |
| 跳转网站 | 小程序内可通过 web-view 跳转网站   |

**技术栈决策说明**：

- 选择原生而非 Taro，因为：
  1. 只需微信小程序，无跨平台需求
  2. 原生更简单，减少框架引入的复杂性
  3. 核心业务在 H5（ysx-core-ui），小程序只是入口壳
- 如未来需要其他平台小程序，再评估迁移方案

## 七、注意事项

### 7.1 二维码稳定性

| 要点             | 说明                                     |
| ---------------- | ---------------------------------------- |
| 域名是核心       | 只要域名不变，二维码永久有效             |
| 证书不影响二维码 | 证书过期只影响访问体验，不影响二维码本身 |
| HTTPS 优先       | 二维码应编码 `https://` 开头的地址       |

### 7.2 SSL 证书续期

- 当前证书有效期至 2026-06-23
- 到期前需续期，建议设置日历提醒
- 续期不影响域名和二维码

### 7.3 小程序码稳定性

- 正式发布后的小程序码永久有效
- 小程序内容可随时更新，码不变
- 体验版/开发版的码不能用于印刷

### 7.4 印刷要求

- 二维码导出时使用**矢量格式**（SVG/PDF）
- 建议尺寸不小于 2cm x 2cm
- 测试扫码成功率
