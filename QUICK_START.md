> 为新对话窗口 / 新协作者准备的快速上手文档

## 5秒了解项目

**元思想**是一个身心成长平台，融合现代简约与中国传统美学。React + Tailwind CSS 开发，最终转换为 **TARO + SCSS**（微信小程序 + H5）。

**视觉关键词**：琥珀金 + 鼠尾草绿 + 古纸底色 + 宋体点睛 + 节气时辰（印章标记规划中）

## 页面架构

「固定首屏 + 内容上滑覆盖」架构：
- Hero 区 `position: fixed` 固定不动
- 内容页从底部自然上滑覆盖 Hero，圆角透出背景图
- Header 吸顶阈值 90vh（内容页接近顶部时过渡为绢轴色吸顶）
- 向下滑动回退时，Header 恢复透明 Hero 样式

## 设计 Tokens

所有样式变量在 `/src/styles/theme.css`：

```css
/* 主色调 */
--color-primary: #C49A6C;        /* 暖琥珀金 */
--color-secondary: #8BAA7D;      /* 鼠尾草绿 */

/* 背景色 */
绢轴色: #F0E4CE → #F5E8D2

/* 字体 */
宋体: 'Noto Serif SC', serif    /* 标题/金句 */

/* 响应式单位 */
--rpx: calc(100vw / 750);       /* 750px 设计基准 */
```

## 重要约束

### 不要做的事
- 不要修改 `/src/app/components/figma/ImageWithFallback.tsx`（受保护）
- 不要删除 `/src/app/components/ui/` 下的 shadcn 组件（受保护）
- 不要创建 `tailwind.config.js`（使用 Tailwind v4）
- 不要在组件里硬编码文本/图片（放在 config 层）
- 不要使用纯黑色遮罩（用深琥珀色 `rgba(30,20,10,...)`）
- 不要反转功能性入口的阅读方向（卡片、导航栏保持现代左→右）

### 设计原则
- 出现 2 次以上 → 做成共享组件
- 用阴影不用边框
- 宋体只用于标题/金句
- 白色文字加三层 text-shadow
- 基于 750px 设计基准
- 功能性 UI → 左→右；氛围装饰 → 可右→左

## 常见任务

### 新建页面
1. 在 `/src/app/pages/` 创建页面组件
2. 内容数据放在 `config/` 目录
3. 使用 `shared/` 下的原子组件构建
4. 页面只负责组装，不含硬编码数据

### 新建组件
1. 判断放在 `shared/`（跨页面复用）还是特定目录
2. Props 设计要灵活，onClick 回调必须预留
3. 添加清晰的 JSDoc 注释
4. 更新 `/COMPONENTS.md`
